# Claude code

## 安装步骤

```md
curl -fsSL https://claude.ai/install.sh | bash
```

## 使用方法

```md
cd /project

终端启动命令
[claude: 当前目录启动 Claude Code 交互会话]
[claude -c: 继续上一次会话]
[claude -p "指令": 非交互直接执行任务]
[claude -r 会话 ID: 恢复指定历史会话]

会话操作斜杠命令
[/help: 查看全部帮助命令]
[/clear: 清空当前对话上下文]
[/compact: 压缩会话历史、省 Token]
[/rewind: 回退到上一步代码状态]
[/exit: 退出 Claude Code 会话]

模型与配置
[/model: 切换 Sonnet/Opus/Haiku 模型]
[/config: 打开全局配置]
[/status: 查看会话、路径、模型状态]
[/cost: 查看当前会话 Token 消耗]
[/doctor: 环境 & 连接问题诊断]
[/effort: 设置思考深度 high/medium/low]

项目记忆 & 任务
[/init: 生成项目 CLAUDE.md 记忆文件]
[/memory: 编辑项目记忆 CLAUDE.md]
[/todos: 管理项目待办任务列表]

快捷工具
[/copy: 复制最近一次回答到剪贴板]
[/export 文件名：导出整段对话存档]
[/add-dir: 关联额外项目目录]
[/permissions: 管理命令执行权限]
[/usage: 查看额度与限流状态
```

## [everything-claude-code](https://github.com/affaan-m/everything-claude-code)

### 安装步骤
```md
# Add marketplace
/plugin marketplace add https://github.com/affaan-m/everything-claude-code

# Install plugin
/plugin install ecc@ecc
```

### 使用方法

```md
核心工作流命令
[/plan:制定功能的详细实现计划，不直接修改代码]
[/tdd:启动测试驱动开发模式，先编写并运行测试再实现功能]
[/code-review:执行全面的代码质量审查并输出改进建议]
[/security-review:全面扫描项目并审查潜在的安全漏洞]
[/refactor:重构指定模块或代码块以提升代码质量与结构]

维护与诊断命令
[/build-fix:在项目构建失败时分析错误日志并实施自动修复]
[/e2e:生成、配置并运行端到端测试用例]
[/performance:针对特定业务场景或数据库查询进行性能优化]
[/cleanup:扫描并清理项目中的死代码及未使用的引用]
[/doctor:诊断当前运行环境、插件依赖以及配置问题]
[/repair:修复损坏的组件、规则目录或意外中断的安装]
[/status:查看当前会话的上下文状态以及令牌等资源指标]
[/learn:从当前的开发会话与修改历史中自动提取复用模式]

并行开发扩展命令（需安装 Worktree 依赖组件）
[/multi-plan:同时为多个独立的任务或分支并行制定实现计划]
[/multi-execute:通过多 Git Worktree 并行执行不同的代码开发任务]
[/multi-backend:在隔离的工作区中同时开发多个后端功能模块]
[/multi-frontend:在隔离的工作区中同时编写或迁移多个前端界面]
[/multi-workflow:触发端到端的多任务并行处理及自动合并流水线]
```
